# 2026-06-26 工作日志

## 一、调试模式研究

### 目标
研究 G1 机器人调试模式（`rt/user_lowcmd`）并调通。

### 调研内容

- 查阅宇树官方文档（https://support.unitree.com/home/zh/G1_developer/）
- 阅读 SDK 源码：`g1_userctrl_dds_example.cpp`、`g1_loco_client_example.cpp`、`g1_ankle_swing_example.cpp` 等
- 分析 LocoClient RPC 机制（`rt/api/sport/request` & `rt/api/sport/response`）
- 理解 G1 双机架构：PC1（运控专用，不开放）+ PC2（开发者使用，IP 192.168.123.164）

### 三种控制方式对比

| | `rt/arm_sdk` | `rt/user_lowcmd` | `rt/lowcmd` |
|---|---|---|---|
| 控制范围 | 全身 29 DOF（weight 混合） | 全身 29 DOF（直接控制） | 全身 29 DOF（最底层） |
| FSM 要求 | 任意状态 | PASSIVE（FSM=1） | 需 ReleaseMode |
| 运控状态 | 继续运行 | 暂停 | 完全释放 |
| CRC32 | 不需要 | 不需要 | 需要 |
| 当前可用 | ✅ | ❌ | 未测试 |

### 遇到的问题

1. **GetFsmId 超时（ret=3104）**
   - 原因：LocoClient 初始化顺序问题
   - 解决：调整为先创建 LocoClient，再创建 publisher/subscriber

2. **FSM ID 返回非标准值**
   - 走跑模式返回 801（ArmAction/Expert）
   - PASSIVE 模式（遥控器）返回 65535（无效值）
   - 解决：添加非标准 ID 识别，跳过 FSM 检查直接尝试 SwitchToUserCtrl

3. **SwitchToUserCtrl 返回 3203（API not implemented）**
   - 从 FSM=801 调用 → ret=3203
   - 从 FSM=1（通过 Damp()）调用 → ret=3203
   - 结论：**该固件版本不支持 SwitchToUserCtrl API（RPC 7110）**

4. **遥控器进入 PASSIVE 后 sport 服务不响应**
   - GetFsmId 超时或返回 65535
   - `rt/user_lowcmd` 电机不响应

### 结论

调试模式在当前固件版本不可用。`SwitchToUserCtrl` API 未实现，`rt/user_lowcmd` 在 PASSIVE 模式下电机也不响应。彻底放弃调试模式开发，专注走跑模式（`rt/arm_sdk`）。

### 输出

- `src/csv_replay_debug.cpp` — 调试模式回放脚本（代码保留，当前不可用）
- `src/g1_mode_switch.cpp` — FSM 状态切换工具

## 二、走跑模式稳定性优化

### 问题

上肢动作改变重心，导致：
- 腰部摆动
- 机器人自动迈步或弯腰补偿

### 解决方案

#### 1. 分关节 PD 增益（按电机类型设置）

参考 `g1_ankle_swing_example.cpp` 中的电机类型和推荐增益：

| 关节 | 电机类型 | Kp | Kd | 说明 |
|------|---------|-----|-----|------|
| 手臂 (15-28) | GearboxS | 60 | 1.5 | 跟踪 CSV 目标 |
| 腰Yaw (12) | GearboxM | 80 | 2 | 中等硬度，抵抗运控补偿 |
| 腰Roll/Pitch (13-14) | GearboxS | 50 | 1.5 | 中等硬度 |
| 髋关节 (0-2, 6-8) | GearboxM | 60 | 1 | 抵抗重心偏移 |
| 膝关节 (3, 9) | GearboxL | 100 | 2 | 强力支撑 |
| 踝关节 (4-5, 10-11) | GearboxS | 40 | 1 | 稳定支撑 |

#### 2. BalanceStand + LowStand

回放前调用：
```cpp
loco.BalanceStand();  // 切换到平衡站立模式，抑制自动迈步
loco.LowStand();      // 降低重心，增大稳定裕度
```

#### 3. 测试效果

| 指标 | 改进前 | 改进后 |
|------|--------|--------|
| 手臂跟踪误差 | ~1.2 rad | ~0.19 rad（改善 6 倍） |
| 腿部偏移 | ~0.00003 rad | ~0.051 rad（稳定不增长） |
| 最差关节 | 肩关节 | 腰Pitch（内置运控补偿） |

### 待解决问题

- 腰部摆动幅度仍较大（内置运控用腰补偿重心）
- 需要进一步调整腰增益或优化动作轨迹

## 四、关键认知

1. **`rt/arm_sdk` 可控制全部 29 个关节**，不限于上肢，名字有误导性
2. **weight 机制是全局的**，weight=1.0 时必须给所有关节发指令（否则腿会收到默认值 0）
3. **调试模式需要固件支持 `SwitchToUserCtrl` API**，当前版本不支持（ret=3203）
4. **分关节增益**比统一增益更有效，应按电机类型（GearboxS/M/L）设置
5. **G1 架构**：PC1（运控，不开放）+ PC2（开发者），sport 服务在 PC1 上
